Codebook.mdVariables in this database came from an experimental set of smart phone accelerometer and gyroscope readings that were collected while subjects were performing known physical activities.  The data had been randomly divided into training and test subsets, and for this project needed to be recombined into a single tidy data set.  The data as received was contained in two sets of three files each (six data files in total).FILESX_test.txt - 2947 observations of feature data in 561 columnsy_test.txt – 2947 data points consisting of the activity code for each row of X_test.txtsubject_test.txt – 2947 subject ID numbers corresponding to the subject being tested in each row of X_test.txtX_train.txt - 7352 observations of feature data in 561 columnsy_train.txt – 7352 data points consisting of the activity code for each row of X_train.txtsubject_train.txt – 7352 subject ID numbers corresponding to the subject being tested in each row of X_train.txtVARIABLESOriginal column numbers and labels from X_test.txt and X_train.txt. columnsSubject_ID – subject identity code (new column #1)Activity – subject activity being measured (new column #2).  Originally a numeric code, transformed to human readable labels where 1 ->  walking, 2 -> walking upstairs, 3 -> walking downstairs, 4 -> sitting, 5 -> standing, 6 -> laying.Keywordstime_domain– indicates time domain datafft_domain – frequency domain signal calculated using Fast Fourier transform of corresponding time databody – body component of acceleration data (high frequency data)gravity – gravity component of acceleration data (low frequency data)Jerk – So-called “jerk” signal calculated from derivative of corresponding time datamagnitude – magnitude of three dimensional signal calculated using the Euclidean normX-axis - x-axis data from gyro or accelerometerY-axis -  y-axis data from gyro or accelerometerZ-axis -  z-axis data from gyro or accelerometermean – mean of signalstd – standard deviation of signalSee: features_info.txt from the UCI HAR data folder for more details about data collection.PROCESSINGProcessing is explained within the fully commented R script: run_analysis.R.
